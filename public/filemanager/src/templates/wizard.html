1. Decentralized content

The SHIFT client will resolve our latest content from the IPFS network.

All files are nested in a tree, so we can keep it structured, like we are all used to.


The content we load here, is from the top level domain: ipfspage.com.


2. Versioning

You are able to modify the content yourself, using the build-in code editor.

There's a help button to assist you. Don't worry.


New: it's even possible to drop a full site folder from you OS onto the file manager.

3. Publishing

Publishing your content to a top level domain is just a one-click process.

So you can publish the content to ipfspage.com yourself.*


* = Sure, not anybody should to be able to modify, but for this demo we'll let you all.

Let's get started

Once you're finished modifying, preview in gateway. Satisfied? Then publish.

Why a regular .com domain? That doesn't look distributed.* Well.. it is!


That's all. Now you are able to go try it out yourself. Have fun!


* = We just want you to see decentral content is made easy :-)
